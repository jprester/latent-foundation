# Latent Foundation

## Overview

Latent Foundation is a project focused on creating and sharing SCP-style stories, artifacts, and anomalies that were generated by LLMs like ChatGPT, Claude, and Gemini. It features automatic thumbnail generation via Fal.ai and supports multiple LLM providers (currently OpenRouter and Anthropic) for story creation.

## How to Install

1. Clone this repository
2. `npm install`
3. Copy `.env.example` to `.env` and add your API keys
4. `npm run dev`

### Environment Variables

| Variable              | Required                 | Description                                                                                                       |
| --------------------- | ------------------------ | ----------------------------------------------------------------------------------------------------------------- |
| `ANTHROPIC_API_KEY`   | For Anthropic provider   | Claude API key from [console.anthropic.com](https://console.anthropic.com/)                                       |
| `OPENROUTER_API_KEY`  | For OpenRouter provider  | API key from [openrouter.ai/keys](https://openrouter.ai/keys)                                                     |
| `FAL_KEY`             | For thumbnail generation | API key from [fal.ai/dashboard/keys](https://fal.ai/dashboard/keys)                                               |
| `OPENROUTER_SITE_URL` | No                       | Optional site URL for OpenRouter referrer header                                                                  |
| `ANTHROPIC_MODEL`     | No                       | Default model id or alias when `--provider anthropic` is used. Defaults to `claude-sonnet-4-6`.                   |
| `OPENROUTER_MODEL`    | No                       | Default model id or alias when OpenRouter is used. Defaults to `moonshotai/kimi-k2.6`.                            |

## Story Generation

If you want to use repo generation tools, there are two ways to generate a story:

1. **CLI script** — single-shot, headless, one command per story. Best for quick generation.
2. **Claude Code agentic flow** — multi-agent pipeline (ideator → writer) with stronger creative output. Best when you want a great story and don't mind opening a Claude Code session. (requires a Claude subscription and claude-code installed)

### Method 1: CLI Script

Stories are generated via a CLI script that supports multiple LLM providers and automatic thumbnail generation.

```bash
# Generate a new story (uses OpenRouter running Kimi K2.6 model by default)
npm run generate -- --theme "haunted mirror" --class "Euclid"

# With additional options
npm run generate -- \
  --theme "sentient AI that writes poetry" \
  --class "Safe" \
  --tags "artificial-intelligence,writing,benevolent" \
  --location "MIT Computer Science Lab" \
  --researcher "Stevens"

# Use Anthropic provider instead (Sonnet 4.6 as default)
npm run generate -- --theme "ancient relic" --class "Keter" --provider anthropic

# Skip thumbnail generation
npm run generate -- --theme "interdimensional doorway" --class "Keter" --no-image

# Longer story (default is ~2000 words, 4000 max tokens)
npm run generate -- --theme "ancient simulated reality machine" --class "Euclid" \
  --provider anthropic --min-words 5000 --max-tokens 16000
```

#### Supported Providers

- **OpenRouter** (default): Defaults to Kimi K2.6 (`moonshotai/kimi-k2.6`). Any other OpenRouter model can be passed via `--model`.
- **Anthropic**: Direct Claude API access. Defaults to Sonnet 4.6.

#### Choosing a model

Three ways to pick a model, in priority order:

1. **`--model` flag** — overrides everything: `--model claude-opus-4-7` or `--model "google/gemini-2.0-flash-exp"`
2. **Env var** in `.env` — `ANTHROPIC_MODEL` or `OPENROUTER_MODEL`. Useful when you always want the same model for a given provider.
3. **Built-in default** — Sonnet 4.6 for Anthropic, Kimi K2.6 for OpenRouter.

Short aliases are also accepted (e.g. `--model kimi_2_6`, `--model claude_4_6_opus`). Run `node scripts/generate-story.mjs --help` for the full alias list.

**Reasoning is disabled on OpenRouter by default.** The script sends `reasoning: { enabled: false }` in the request body, which tells reasoning-capable models (Kimi K2.6, DeepSeek R1, etc.) to skip the hidden "thinking" step and just answer. Without this flag, those models burn through `max_tokens` on hidden reasoning and never emit the actual story. If you want a model to reason and have the budget for it, bump `--max-tokens` to `32000`+ and flip the `reasoning.enabled` flag in `scripts/generate-story.mjs:34`.

### Method 2: Claude Code (`/scpg`)

If you use [Claude Code](https://claude.com/claude-code), the project ships with a `/scpg` slash command that runs a multi-agent generation pipeline against your Claude subscription instead of paying per-token via the API. Output quality is meaningfully higher because the writer agent gets a sharpened premise from a dedicated ideator agent.

```bash
claude                          # start a session in the project root
> /model opus                   # the writer benefits noticeably from Opus
> /scpg elevator that opens onto conversations you never had
```

You can pass a class hint, or invoke it bare to let the ideator pick freely:

```
> /scpg                                              # full surprise
> /scpg haunted lighthouse                           # theme, class chosen for you
> /scpg haunted lighthouse, keter                    # theme + class
```

The command:

1. Computes the next `SCPG-NNN` number
2. Invokes the `scp-ideator` subagent to sharpen the premise
3. Invokes the `scp-writer` subagent to draft the entry
4. Saves `stories/scpg-NNN.md` with correct frontmatter
5. Shells out to the existing CLI script for the Fal.ai thumbnail

The pipeline lives in `.claude/`:

- `commands/scpg.md` — the slash command
- `skills/generate-scpg/SKILL.md` — orchestration rules (format, frontmatter, file naming)
- `agents/scp-ideator.md` — premise generator
- `agents/scp-writer.md` — entry drafter

You still need `FAL_KEY` in `.env` for the thumbnail step.

### SCP Classifications

- **Safe**: Easily contained, predictable
- **Euclid**: Unpredictable behavior, requires careful handling
- **Keter**: Extremely dangerous, difficult to contain
- **Apollyon**: Unstoppable, uncontainable, world-ending potential

## Thumbnail Generation

Thumbnails are automatically generated via [Fal.ai](https://fal.ai) using Flux image models. The image prompt is derived from the story's title, class, and content.

```bash
# Generate missing thumbnails for all stories
npm run generate:thumbnails

# Generate thumbnail for a specific story
npm run generate -- --generate-thumbnails --story scpg-031

# Regenerate an existing thumbnail
npm run generate -- --generate-thumbnails --story 15 --force

# Use higher quality model
npm run generate -- --generate-thumbnails --fal-model "fal-ai/flux-pro"
```

Thumbnails are saved to `public/images/[story-id]/thumbnail.jpg`.

## CLI Reference

```
node scripts/generate-story.mjs --theme "THEME" --class "CLASS" [OPTIONS]
node scripts/generate-story.mjs --generate-thumbnails [OPTIONS]
```

| Flag                    | Type   | Description                                                                 |
| ----------------------- | ------ | --------------------------------------------------------------------------- |
| `--theme`               | string | Story theme/concept (required for story generation)                         |
| `--class`               | string | SCP class: Safe, Euclid, Keter, or Apollyon (required for story generation) |
| `--tags`                | csv    | Comma-separated tags                                                        |
| `--location`            | string | Discovery/containment location                                              |
| `--researcher`          | string | Researcher name                                                             |
| `--thumbnail`           | string | Manual thumbnail filename (skips Fal.ai generation)                         |
| `--no-image`            | flag   | Skip thumbnail generation entirely                                          |
| `--images`              | csv    | Comma-separated additional image files                                      |
| `--number`              | number | Specific SCP number (auto-generated if omitted)                             |
| `--provider`            | string | `openrouter` (default) or `anthropic`                                       |
| `--model`               | string | Model ID or alias for selected provider                                     |
| `--max-tokens`          | number | Max output tokens (default `4000`; raise to `16000`+ for longer stories)    |
| `--min-words`           | number | Target minimum word count embedded in the prompt (default `2000`)           |
| `--fal-model`           | string | Fal.ai model: `fal-ai/flux/dev` (default) or `fal-ai/flux-pro`              |
| `--generate-thumbnails` | flag   | Backfill thumbnails for existing stories                                    |
| `--story`               | string | Target specific story (e.g., `scpg-031` or `31`)                            |
| `--force`               | flag   | Regenerate thumbnail even if it already exists                              |

## How to Contribute

Clone this repository, create a new Markdown file in the `stories` directory, and follow the SCP format. Use the provided examples as a guide, and then submit a pull request with your additions. Of course, you an also do your own thing and generate stories via the CLI or Claude Code flow, then submit PRs with those!
